Code Review

This walkthrough highlights the C++ logic, data validation layers, and how external testing methods were used to identify vulnerabilities. Runtime analysis and enhancements for BST tree balancing and error handling are also explained.

Course Navigator Pro Code Review

Original Artifact

This artifact began as a course requirement at ABC University. The original application was built in C++ and designed to read a CSV of course data, organize it using a simple Binary Search Tree (BST), and allow users to view courses in sorted order or fetch prerequisites.

While the core concept demonstrated an understanding of BST logic, the app lacked scalability, edge case handling, and modern data persistence. It also assumed perfectly formatted input files, limiting its real-world application.

Original Console App in C++ w/ BST Sorting

Enhancement

For my enhancement, I performed a complete overhaul of the architecture. The entire codebase was ported from C++ to Java to enable cross-platform use and better code modularity.

The original BST was replaced with a self-balancing AVL Tree, which improves performance in cases of heavy insertions or searches. This restructuring ensures that lookups, inserts, and traversals remain efficient even with thousands of records.

Additionally, a MongoDB database was introduced to manage persistent course data and allow for expanded record structures (such as multiple prerequisites, department metadata, etc.). These enhancements prepare the application for a full-stack transition.

Improved Architecture with Java, AVL Trees, and MongoDB

Future Enhancements

Looking ahead, I plan to build a web-based interface that connects to the backend using RESTful APIs. This UI would allow advisors to perform course searches, view prerequisites, and export advising reports in real-time.

Future goals also include authentication features for advisors, role-based access controls, and deploying the system to a cloud environment such as Firebase or AWS for full scalability and availability.
